# 📖 Guia de Uso - Servidores MCP

## Visão Geral

Este sistema é composto por dois servidores MCP que se comunicam via JSON-RPC sobre HTTP.

### Servidor 1: configuracao-ddl-mcp (porta 8081)

Responsável por:
- Converter scripts DDL em classes Entity JPA
- Identificar tabelas/colunas não reconhecidas
- Recompilar o projeto de entidades

### Servidor 2: metadados-massa-mcp (porta 8082)

Responsável por:
- Retornar metadados de entidades JPA
- Interpretar consultas em linguagem natural
- Criar massa de dados estruturada
- Gerenciar demandas de configuração

---

## 🔧 Configuração

### application.properties (configuracao-ddl-mcp)

```properties
server.port=8081
entity.classes.path=C:/caminho/para/entities-core/src/main/java
entity.project.path=C:/caminho/para/metadados-massa-mcp
entity.core.project.path=C:/caminho/para/entities-core
```

### application.properties (metadados-massa-mcp)

```properties
server.port=8082
entity.classes.path=C:/caminho/para/entities-core/src/main/java
demandas.path=C:/caminho/para/metadados-massa-mcp/demandas
```

### .continue/config.json

```json
{
  "mcpServers": {
    "configuracao-ddl-mcp": {
      "type": "http",
      "url": "http://localhost:8081/mcp"
    },
    "metadados-massa-mcp": {
      "type": "http",
      "url": "http://localhost:8082/mcp"
    }
  }
}
```

## 🛠️ Ferramentas MCP

### configuracao-ddl-mcp

#### `ddl_to_entity`

Converte um script DDL em uma classe Entity JPA.

**Parâmetros:**
- `ddlScript` (string, obrigatório): Script SQL com CREATE TABLE

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "ddl_to_entity",
    "arguments": {
      "ddlScript": "CREATE TABLE T696FORU (SQ_FRU int GENERATED BY DEFAULT AS IDENTITY NOT NULL, NM_FRU varchar(100) NOT NULL, CONSTRAINT PK_FORU PRIMARY KEY (SQ_FRU));"
    }
  },
  "id": 1
}
```

**Resposta:**
```json
{
  "jsonrpc": "2.0",
  "result": {
    "content": [{
      "type": "text",
      "text": "{\"entityClassName\":\"Foru\",\"tableName\":\"T696FORU\",\"compileSuccess\":true}"
    }]
  },
  "id": 1
}
```

#### `identify_unknown_entities`

Compara tabelas/colunas do DDL com as classes Entity existentes.

**Parâmetros:**
- `ddlScript` (string, obrigatório): Script DDL para análise

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "identify_unknown_entities",
    "arguments": {
      "ddlScript": "CREATE TABLE T696NOVA (SQ_NOVA int IDENTITY, NM_NOVA varchar(100));"
    }
  },
  "id": 1
}
```

**Resposta:**
```json
{
  "jsonrpc": "2.0",
  "result": {
    "content": [{
      "type": "text",
      "text": "{\"missingTables\":[\"T696NOVA\"],\"missingColumns\":[]}"
    }]
  },
  "id": 1
}
```

### metadados-massa-mcp

#### `get_entity_metadata`

Retorna metadados de uma classe Entity JPA.

**Parâmetros:**
- `className` (string, obrigatório): Nome completo da classe

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "get_entity_metadata",
    "arguments": {
      "className": "br.gov.bnb.domain.entity.Portfolio"
    }
  },
  "id": 1
}
```

**Resposta:**
```json
{
  "jsonrpc": "2.0",
  "result": {
    "content": [{
      "type": "text",
      "text": "{\"className\":\"Portfolio\",\"tableName\":\"T696POAC\",\"attributes\":[...],\"relationships\":[...]}"
    }]
  },
  "id": 1
}
```

#### `get_mapeamento_semantico`

Retorna o mapeamento semântico completo (entidades, atributos, sinônimos).

**Parâmetros:** Nenhum

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "get_mapeamento_semantico",
    "arguments": {}
  },
  "id": 1
}
```

#### `consultar_dados`

Executa uma consulta estruturada no banco de dados.

**Parâmetros:**
- `comando` (string, obrigatório): "consultar"
- `entidade` (string, obrigatório): Nome da entidade no mapeamento semântico
- `filtros` (array, opcional): Lista de filtros
- `agregacao` (string, opcional): count, sum, avg, min, max
- `ordenacao` (object, opcional): Atributo e direção
- `limite` (integer, opcional): Limite de resultados

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "consultar_dados",
    "arguments": {
      "comando": "consultar",
      "entidade": "acao_estrategica",
      "filtros": [{"atributo": "situacao", "operador": "=", "valor": "Ativo"}],
      "agregacao": "count"
    }
  },
  "id": 1
}
```

#### `criar_massa`

Cria massa de dados no banco a partir de um JSON estruturado.

**Parâmetros:**
- `entidades` (array, obrigatório): Lista de entidades a serem criadas

**Exemplo:**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "criar_massa",
    "arguments": {
      "entidades": [{
        "tipo": "portfolio",
        "dados": {"nome": "Portfolio Teste", "ano_realizacao": 2026, "situacao": "Em Elaboração"},
        "filhos": [{
          "tipo": "acao_estrategica",
          "dados": {"nome": "Ação 1", "descricao": "Descrição", "situacao": "Proposta em Edição"}
        }]
      }]
    }
  },
  "id": 1
}
```

#### `listar_demandas`

Lista as demandas pendentes de configuração.

**Parâmetros:** Nenhum

#### `resolver_demanda`

Marca uma demanda como resolvida.

**Parâmetros:**
- `id` (string, obrigatório): ID da demanda (nome do arquivo)

#### `detalhar_demanda`

Retorna detalhes completos de uma demanda específica.

**Parâmetros:**
- `id` (string, obrigatório): ID da demanda (nome do arquivo)

## 🔍 Resolução de Problemas

### Servidor não inicia

1. Verificar se Java 21 está instalado: `java --version`
2. Verificar se Maven está instalado: `mvn --version`
3. Verificar se as portas 8081 e 8082 estão livres
4. Verificar se os caminhos no `application.properties` existem

### Ferramenta retorna erro -32602

- Verificar se os parâmetros obrigatórios foram fornecidos
- Verificar se o DDL tem sintaxe SQL válida
- Verificar se o nome da classe existe no classpath

### Compilação falha

- Verificar se o `entity.core.project.path` aponta para o diretório correto
- Verificar se o Maven está configurado corretamente
- Verificar se não há erros de sintaxe no código gerado

### Consulta não retorna dados

- Verificar se o banco H2 tem dados (usar console H2 em `/h2-console`)
- Verificar se o mapeamento semântico está correto
- Verificar se a entidade existe no classpath

## 📝 Exemplos Práticos

### Exemplo 1: Criar uma nova entidade a partir de DDL

```bash
curl -X POST http://localhost:8081/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "ddl_to_entity",
      "arguments": {
        "ddlScript": "CREATE TABLE T696FORU (SQ_FRU int GENERATED BY DEFAULT AS IDENTITY NOT NULL, NM_FRU varchar(100) NOT NULL, CONSTRAINT PK_FORU PRIMARY KEY (SQ_FRU));"
      }
    },
    "id": 1
  }'
```
