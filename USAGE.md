# 📖 Guia de Uso — Massa Dados Core

## Configuração

### application.properties

```properties
# Porta do servidor
server.port=8080

# Nome do servidor MCP
mcp.server.name=massa-dados-core

# Versão do servidor MCP
mcp.server.version=1.0.0

# Caminho das classes Entity (onde serão salvas as classes geradas)
entity.classes.path=src/main/java/com/thadeu/massa-dados-core/domain

# Caminho do projeto (para recompilação)
entity.project.path=.
```

### Configuração do Continue

Adicione no `~/.continue/config.json` (ou `config.ts`):

```json
{
  "experimental": {
    "modelContextProtocolServers": [
      {
        "name": "massa-dados-core",
        "url": "http://localhost:8080/mcp"
      }
    ]
  }
}
```

## Ferramentas MCP

### 1. `ddl_to_entity`

Converte um script DDL em classes Entity JPA.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `ddlScript` | string | Sim | Script DDL contendo CREATE TABLE, ALTER TABLE, etc. |

**Exemplo de uso no chat:**

```
Use a ferramenta ddl_to_entity com o seguinte DDL:

CREATE TABLE T696CLIENTE (
    SQ_CLI int GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    NM_CLI varchar(100) NOT NULL,
    CONSTRAINT PK_CLIENTE PRIMARY KEY (SQ_CLI)
);
```

**Resposta esperada:**

```json
[
  {
    "entityClassName": "com.thadeu.massa_dados_core.domain.Cliente",
    "entityCode": "package com.thadeu.massa_dados_core.domain;\n\nimport jakarta.persistence.*;\nimport lombok.*;\n\n@Entity\n@Table(name = \"T696CLIENTE\")\n@Data\n@NoArgsConstructor\n@AllArgsConstructor\npublic class Cliente {\n\n    @Column(name = \"SQ_CLI\", nullable = false)\n    @Id\n    @GeneratedValue(strategy = GenerationType.IDENTITY)\n    private Integer sqCli;\n\n    @Column(name = \"NM_CLI\", nullable = false)\n    private String nmCli;\n\n}\n",

      {
```

### 2. `identify_unknown_entities`

Identifica tabelas/colunas no DDL que não possuem classe Entity correspondente.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `ddlScript` | string | Sim | Script DDL contendo CREATE TABLE, ALTER TABLE, etc. |

**Exemplo de uso no chat:**

```
Use a ferramenta identify_unknown_entities com o seguinte DDL:

CREATE TABLE T696NOVATABELA (
    SQ_NOV int GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    NM_NOV varchar(100) NOT NULL,
    CONSTRAINT PK_NOV PRIMARY KEY (SQ_NOV)
);
```

**Resposta esperada:**

```json
{
  "missingTables": ["T696NOVATABELA"],
  "missingColumns": []
}
```

### 3. `get_entity_metadata` (em desenvolvimento)

Obtém metadados de uma entidade JPA via reflection.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `className` | string | Sim | Nome completo da classe (ex: `com.thadeu.massa_dados_core.domain.Cliente`) |

## Resolução de Problemas

### Erro: "Nenhuma tabela encontrada no DDL"

Verifique se o script DDL contém pelo menos um `CREATE TABLE` com a sintaxe correta.

### Erro: "Falha na compilação do Servidor 2"

Verifique se:
- O Maven está instalado e acessível via linha de comando
- O diretório `entity.project.path` está configurado corretamente
- O `pom.xml` do projeto é válido

### Erro: "Classe não encontrada"

Verifique se:
- A classe foi gerada corretamente no diretório `entity.classes.path`
- O servidor foi recompilado após a geração
- O nome da classe está correto (incluindo o pacote)
