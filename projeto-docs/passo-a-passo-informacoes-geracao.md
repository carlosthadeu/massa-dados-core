# 🧭 Passo a Passo — Informações de Geração de Entidades JPA

> **Servidor responsável:** `configuracao-ddl-mcp` (porta 8081)
> **Módulo alvo:** `entities-core`
> **Pacote das entidades geradas:** `com.thadeu.entities-core.domain.entity`
> **Última atualização:** 2026-07-08

---

## 1. Visão Geral

Este documento descreve como consumir os serviços MCP do servidor `configuracao-ddl-mcp` para obter informações sobre a geração de entidades JPA a partir de scripts DDL.

### Fluxo recomendado

```
1. Obter configuração de geração → get_generation_config
2. Simular geração sem salvar   → simulate_ddl_generation
3. Analisar quadro comparativo   → diffReport na resposta
4. Gerar entidade com relatório  → ddl_to_entity
5. Atualizar mapa de classes     → mapas-classes.md
```

---

## 2. Serviços Disponíveis

### 2.1 `get_generation_config`

Retorna as configurações atuais de geração de entidades (path, pacote, etc.).

**Parâmetros:** Nenhum

**Exemplo de requisição:**

```bash
curl -X POST http://localhost:8081/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "get_generation_config",
      "arguments": {}
    },
    "id": 1
  }'
```

**Exemplo de resposta:**

```json
{
  "jsonrpc": "2.0",
  "result": {
    "content": [{
      "type": "text",
      "text": "{
        \"entityOutputRelativePath\": \"entities-core/src/main/java/com/thadeu/entities-core/domain/entity/\",
        \"packageName\": \"com.thadeu.entities-core.domain.entity\",
        \"sourceEntityPackage\": \"domain.entity\",
        \"sourceEntityRelativePath\": \"entities-core/src/main/java/domain/entity/\",
        \"existingGeneratedEntities\": [],
        \"totalExistingEntities\": 21
      }"
    }]
  },
  "id": 1
}
```

**Campos da resposta:**

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `entityOutputRelativePath` | string | Caminho relativo (a partir da raiz do repositório) onde as entidades geradas são salvas |
| `packageName` | string | Pacote Java das entidades geradas |
| `sourceEntityPackage` | string | Pacote das entidades existentes (manuais) |
| `sourceEntityRelativePath` | string | Caminho relativo das entidades existentes |
| `existingGeneratedEntities` | array | Lista de entidades já geradas no diretório de saída |
| `totalExistingEntities` | integer | Total de entidades manuais existentes |

### 2.2 `simulate_ddl_generation`

Simula a geração de uma entidade a partir de um script DDL **sem salvar nada em disco** (dry-run). Retorna o código que seria gerado e o quadro comparativo.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `ddlScript` | string | Sim | Script SQL contendo instrução CREATE TABLE |

**Exemplo de requisição:**

```bash
curl -X POST http://localhost:8081/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "simulate_ddl_generation",
      "arguments": {
        "ddlScript": "CREATE TABLE T696FORU (SQ_FRU int GENERATED BY DEFAULT AS IDENTITY NOT NULL, NM_FRU varchar(100) NOT NULL, CONSTRAINT PK_FORU PRIMARY KEY (SQ_FRU));"
      }
    },
    "id": 1
  }'
```

**Exemplo de resposta:**

```json
{
  "jsonrpc": "2.0",
  "result": {
    "content": [{
      "type": "text",
      "text": "{
        \"entityClassName\": \"Foru\",
        \"tableName\": \"T696FORU\",
        \"packageName\": \"com.thadeu.entities-core.domain.entity\",
        \"entityOutputRelativePath\": \"entities-core/src/main/java/com/thadeu/entities-core/domain/entity/\",
        \"entityCode\": \"package com.thadeu.entities-core.domain.entity;\\n\\nimport javax.persistence.*;\\n...\\n\",
        \"attributes\": [
          {
            \"name\": \"sqFru\",
            \"type\": \"Long\",
            \"nullable\": false,
            \"primaryKey\": true,
            \"columnName\": \"SQ_FRU\"
          },
          {
            \"name\": \"nmFru\",
            \"type\": \"String\",
            \"nullable\": false,
            \"primaryKey\": false,
            \"columnName\": \"NM_FRU\"
          }
        ],
        \"diffReport\": {
          \"created\": [\"Foru\"],
          \"updated\": [],
          \"unchanged\": [\"Entrega\", \"Plano\", \"Portfolio\", ...],
          \"removed\": []
        },
        \"wouldOverwrite\": false
      }"
    }]
  },
  "id": 1
}
```

**Campos da resposta:**

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `entityClassName` | string | Nome da classe que seria gerada |
| `tableName` | string | Nome da tabela no banco |
| `packageName` | string | Pacote onde seria salva |
| `entityOutputRelativePath` | string | Caminho relativo do arquivo |
| `entityCode` | string | Código Java completo da Entity (escapado) |
| `attributes` | array | Lista de atributos com tipo, coluna, nullable, PK |
| `diffReport` | object | Quadro comparativo (ver seção 3) |
| `wouldOverwrite` | boolean | `true` se já existe uma entidade com o mesmo nome |

## 3. Quadro Comparativo (`diffReport`)

O quadro comparativo é retornado tanto na simulação quanto na geração real. Ele informa o impacto da operação sobre as entidades existentes.

### Estrutura

```json
{
  "diffReport": {
    "created": ["Foru", "OutraTabela"],
    "updated": ["EntidadeExistente"],
    "unchanged": ["Entrega", "Plano", "Portfolio", ...],
    "removed": []
  }
}
```

| Campo | Significado |
|-------|-------------|
| `created` | Entidades **novas** que serão criadas (não existiam antes) |
| `updated` | Entidades **existentes** que serão sobrescritas |
| `unchanged` | Entidades **existentes** que não serão afetadas |
| `removed` | Entidades que deixarão de existir (nunca ocorre automaticamente) |

### Como interpretar

- **➕ Criada:** Nova tabela mapeada. Deve ser adicionada ao mapa de classes.
- **🛠️ Atualizada:** Entidade sobrescrita. Verificar se a compilação foi bem-sucedida.
- **✅ Mantida:** Nenhuma ação necessária.
- **❌ Removida:** Requer ação manual (o sistema não apaga arquivos automaticamente).

## 4. Manutenção do Mapa de Classes

Após cada geração bem-sucedida, o arquivo `projeto-docs/mapas-classes.md` deve ser atualizado.

### 4.1 Seções no mapa de classes

Atualmente o mapa possui:

- **domain.entity** — Entidades manuais (pacote `domain.entity`)
- **domain.enums** — Enums existentes
- **domain.enums.converters** — Conversores JPA

**Adicionar seção para entidades geradas:**

```markdown
## com.thadeu.entities-core.domain.entity

| Classe | Tabela | Arquivo |
|--------|--------|---------|
| `Foru` | `T696FORU` | `com/thadeu/entities-core/domain/entity/Foru.java` |
```

### 4.2 Quando atualizar

- Após executar `ddl_to_entity` com sucesso
- Quando uma entidade for removida manualmente
- Quando houver alteração no pacote ou caminho de saída

## 6. Resolução de Problemas

| Problema | Causa provável | Solução |
|----------|---------------|---------|
| `get_generation_config` retorna `existingGeneratedEntities` vazio | Nenhuma entidade foi gerada ainda pelo sistema automático | Verificar se o diretório de saída existe |
| `simulate_ddl_generation` retorna erro de parsing | DDL mal formatado | Verificar se o DDL contém `CREATE TABLE` válido |
| `wouldOverwrite` = `true` no simulate | Já existe uma entidade com o mesmo nome | Revisar se deseja sobrescrever ou alterar o DDL |
| Compilação falha após `ddl_to_entity` | Erro no código gerado ou dependências | Verificar `compileMessage` para detalhes |
| Path não encontrado | Configuração incorreta no `application.properties` | Verificar `entity.classes.path` |

"## 7. Correções Necessárias no Servidor

### 7.1 Escopo do `identify_unknown_entities` apontando para o local errado

**Problema identificado:**

O método `getExistingTables()` no `McpUnknownEntityService` está escaneando o diretório:

```
entity.classes.path + /br/gov/bnb/domain/entity/
```

Que corresponde a:

```
entities-core/src/main/java/domain/br/gov/bnb/domain/entity/
```

Porém as entidades manuais existentes estão em:

```
entities-core/src/main/java/domain/entity/   (pacote domain.entity)
```

Ou seja, o serviço está procurando no **diretório de saída das entidades geradas**, não no **diretório das entidades manuais**. Por isso, ao executar `identify_unknown_entities` com um DDL de uma tabela que já existe (ex.: `T696ACAO`), ele retorna como "não encontrada" — mesmo que a entidade correspondente já exista no pacote `domain.entity`.

**Solução necessária:**

O `McpUnknownEntityService` deve escanear **ambos** os diretórios:

1. **Diretório das entidades manuais:** `entity.classes.path + /domain/entity/` (pacote `domain.entity`)
2. **Diretório das entidades geradas:** `entity.classes.path + /com/thadeu/entities-core/domain/entity/` (pacote `com.thadeu.entities-core.domain.entity`)

**Alteração no código (`McpUnknownEntityService.java`):**

```java
private List<String> getExistingTables() throws IOException {
    List<String> allTables = new ArrayList<>();

    // 1. Escanear entidades manuais (domain.entity)
    Path manualDir = Paths.get(entityClassesPath, "domain", "entity");
    if (Files.exists(manualDir)) {
        try (Stream<Path> files = Files.list(manualDir)) {
            files.filter(p -> p.toString().endsWith(".java"))
                 .map(this::extractTableNameFromFile)
                 .filter(Objects::nonNull)
                 .forEach(allTables::add);
        }
    }

    // 2. Escanear entidades geradas (com.thadeu.entities-core.domain.entity)
    Path generatedDir = Paths.get(entityClassesPath, "com", "thadeu", "entities-core", "domain", "entity");
    if (Files.exists(generatedDir)) {
        try (Stream<Path> files = Files.list(generatedDir)) {
            files.filter(p -> p.toString().endsWith(".java"))
                 .map(this::extractTableNameFromFile)
                 .filter(Objects::nonNull)
                 .forEach(allTables::add);
        }
    }

    return allTables;
}
```

O mesmo ajuste deve ser aplicado no método `getExistingColumns()`.

**Impacto:** Após a correção, o `identify_unknown_entities` passará a reconhecer corretamente as tabelas mapeadas no pacote `domain.entity`, evitando falsos positivos de "tabela não encontrada".
